# Proyecto Appi Precios

## Descripción
Este proyecto fue desarrollado como parte de una actividad académica orientada a comprender la gestión de proyectos colaborativos utilizando Git y GitHub.
El objetivo principal es aplicar conceptos fundamentales como el control de versiones, la creación de ramas, la realización de commits y la sincronización entre 
repositorios locales y remotos. Asimismo, se busca evidenciar el trabajo en equipo mediante la colaboración entre distintos integrantes dentro de un mismo repositorio.

## Cómo clonar el repositorio
Para obtener una copia local del proyecto, ejecuta el siguiente comando en tu terminal:
git clone https://github.com/kevinperez26/appi_precios.git
Luego accede a la carpeta del proyecto:
cd appi_precios

## Flujo de trabajo colaborativo
El desarrollo del proyecto se realizó utilizando una estructura basada en ramas, específicamente la rama:

desarrollo-caracteristica

En esta rama se realizaron los cambios y mejoras del proyecto. Cada integrante contribuyó mediante commits, permitiendo llevar un control del historial de 
modificaciones.

## Cómo contribuir
Clonar el repositorio
Ubicarse en la rama de trabajo
Realizar modificaciones en los archivos
Guardar los cambios con un commit
Sincronizar los cambios con el repositorio remoto

## Sincronización de cambios
Para actualizar el repositorio local con los cambios realizados en GitHub, se utiliza el comando:

git pull origin desarrollo-caracteristica

Este comando permite mantener actualizado el proyecto y asegurar que todos los integrantes trabajen con la misma versión.

## Conclusión
Este proyecto permitió comprender la importancia del uso de herramientas de control de versiones en entornos colaborativos. Git y GitHub facilitan la organización 
del trabajo, el seguimiento de cambios y la integración de múltiples aportes en un mismo proyecto, convirtiéndose en herramientas esenciales en el desarrollo de software moderno.
